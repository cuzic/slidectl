# GREEN Phase - 最小実装

TDDサイクルの GREEN フェーズ：テストをパスする最小限の実装を行います。

## このフェーズで行うこと

1. **実装ファイルの作成**
   - `src/slidectl/` に適切なモジュールを作成
   - 必要なインポートを追加
   - クラス・関数の骨組みを作成

2. **最小限の実装**
   - テストをパスするために必要な最小限のコードを書く
   - 過剰な実装はしない（YAGNI原則）
   - まずは動作させることを優先

3. **テストの実行とパス確認**
   - `mise run test` を実行
   - すべてのテストがパスすることを確認
   - 新しいテストも既存のテストも通ることを確認

## ベストプラクティス

- **KISS**: Keep It Simple, Stupid - シンプルに保つ
- **YAGNI**: You Aren't Gonna Need It - 必要になってから実装
- **最小限**: テストをパスする最小限の実装のみ
- **動作優先**: 最適化や美しさは後回し

## 実装の流れ

1. 最もシンプルな実装から始める
2. テストを実行
3. 失敗したら修正
4. すべてのテストがパスするまで繰り返す

## 確認事項

- [ ] 実装ファイルが作成された
- [ ] 最小限の実装が完了した
- [ ] `mise run test` を実行した
- [ ] すべてのテストがパスした
- [ ] 既存のテストも壊れていない

## 次のステップ

すべてのテストがパスしたら、REFACTOR フェーズ（リファクタリング）に進みます。
